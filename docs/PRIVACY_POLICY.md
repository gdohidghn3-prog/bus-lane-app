# 개인정보처리방침

**시행일자: 2026년 5월 10일**
**개정일자: 2026년 5월 10일 (위치정보 처리 설명 명확화)**

「버스전용차로」(이하 "본 앱")는 이용자의 개인정보를 중요하게 생각하며, 「개인정보 보호법」 및 「위치정보의 보호 및 이용 등에 관한 법률」을 준수합니다. 본 방침은 본 앱이 어떤 정보를 수집하고 어떻게 사용하는지를 설명합니다.

## 1. 수집하는 정보

| 항목 | 수집 시점 | 처리 위치 | 영구 저장 여부 | 목적 |
|---|---|---|---|---|
| **위치 정보 (실시간 GPS 좌표, 위·경도)** | 앱 사용 중·백그라운드 알림 활성 시 | 단말 → 서버(처리) | **서버에 영구 저장하지 않음** | 주변 버스전용차로 구간 조회 및 접근 경고 판단 |
| **익명화된 경고 로그 (위치 그리드)** | 접근 경고 알림이 발생한 시점 | 서버 DB (Supabase) | 90일 후 자동 삭제 | 서비스 품질 개선 및 오작동 분석 (소수점 3자리·약 100m 격자로 익명화하여 개인 식별 불가) |
| **차량 종류 설정** | 사용자가 직접 선택 시 | 단말 로컬 저장소 | 앱 삭제 시까지 | 차종별 예외(택시, 9인승 이상 등) 적용 |
| **앱 버전·OS 정보·서버 액세스 로그** | 자동 | 서버 로그 | 30일 후 자동 삭제 | 호환성 및 장애 분석 |

본 앱은 **이름·전화번호·이메일·주민등록번호 등 개인 식별 정보를 수집하지 않습니다.** 회원가입이 없는 익명 서비스입니다.

## 2. 위치 정보 처리 원칙 (위치정보보호법 제18조 관련)

- **위치 정보의 서버 전송**: 주변 구간을 조회하기 위해 GPS 좌표(위·경도)는 본 앱 백엔드 서버로 **요청 시점에 일시적으로 전송**됩니다. 서버는 응답을 만들기 위해 좌표를 사용한 직후 폐기하며, **원본 좌표를 영구 저장하지 않습니다.**
- **서버 액세스 로그**: 운영 점검을 위한 액세스 로그에 좌표가 기록될 때는 **소수점 2자리(약 1km 정확도)로 자동 마스킹**되어 개인 식별이 불가능합니다.
- **경고 로그 익명화**: 접근 경고가 발생해 분석용으로 저장되는 로그는 **소수점 3자리(약 100m 격자)로 익명화**되어 저장되며 90일 후 자동 삭제됩니다.
- **백그라운드 위치 권한**: 앱이 닫힌 상태에서도 단속 시간대에 버스전용차로 접근 경고를 제공하기 위해서만 사용됩니다. 본 권한은 **첫 실행 시 온보딩 화면에서 사용자가 명시 동의한 경우에만** 요청·사용되며, 동의하지 않거나 권한을 거부해도 앱의 핵심 기능(지도 표시·구간 조회)은 사용 가능합니다.
- **광고·마케팅·제3자 제공 없음**: 위치 데이터는 광고·마케팅 식별·제3자 제공 목적으로 사용되지 않습니다.
- **전송 보안**: 모든 통신은 HTTPS(TLS 1.2+)로 암호화됩니다.

| 단말 로컬 설정 (차종·도시·알림 토글·온보딩 동의 기록) | 앱 삭제 시까지 |

## 5. 처리 위탁

- **Supabase Inc.** — 데이터베이스 호스팅 (서비스 운영 목적)
- **Render Inc.** — 백엔드 서버 호스팅 (서비스 운영 목적)
- **Google LLC (Google Play)** — 앱 배포

위탁 업체에는 본 방침 1·2·3항에 명시된 정보(원본 좌표 포함, 단 영구 저장은 익명화 후에만)만 전달되며, 위탁 업체는 자체 정책에 따라 정보를 처리합니다.

## 6. 이용자의 권리

이용자는 다음 권리를 행사할 수 있습니다.
- **위치 권한 철회**: 단말 설정 → 앱 → 권한에서 즉시 철회 가능
- **접근 경고 알림 끄기**: 앱 내 설정 → 알림 토글로 즉시 백그라운드 위치 추적 중지
- **로컬 데이터 삭제**: 앱 삭제 시 모든 로컬 데이터가 삭제됩니다.
- **문의·열람 요청**: 아래 연락처로 문의 시 합리적 기간 내 응답합니다.

## 7. 보안 조치

- 모든 통신은 HTTPS(TLS 1.2+)로 암호화됩니다.
- API 인증 키는 환경변수로 분리 관리되며, 서버 액세스 로그의 좌표는 자동 마스킹됩니다.
- 데이터베이스는 Row-Level Security(RLS) 정책으로 접근이 제어됩니다.
- 분석용 경고 로그는 90일 보관 정책에 따라 자동 삭제됩니다(pg_cron 기반).
